GitPerusteet

Git on hajautettu versionhallintajärjestelmä. Linus Torvalds loi alkuperäisen Git'in 2005 Linux -ytimen versionhallintaa varten.

Git on täysin lokaali, mutta git repository voidaan kloonata ja tyypillisesti kloonataan. Repossa on tuotantohaara ja kehityshaara/-haaroja.

Git'in käyttöönotto

Git'iä käytetään seuraavasti: 0) lataa asennusohjelma 1) asenna 2) konfiguroi 3) käytä

Git on maksuton työkalu

Git'in käyttö

Git'in käytetään yleensä seuraavasti: a) perusta projekti omaan hakemistoonsa b) luo projektille git repo (git repository) c) luo tiedostot työalueelle (working area) d) lisää tiedostoja staging -alueelle (staging area) e) lisää staging -alueen tiedostot git repoon (git repository)

Git repon kloonaus

Git repository kloonataan seuraavasti: git clone https//repo.sitory.net/tarkennus/repo.git

Uusien piirteiden lisäys sovellukseen

Git repossa voi ja pitää olla oksia tai haaroja (branch) uusien piirteiden luomista varten. Hyvä git workflow toimii seuraavasti: i) luo haara uudelle piirteelle ii) työstä piirrettä, testaa, korjaa iii) liitä uusi, testattu piirre tuotantohaaraan iv) kesken piirteen tuottamisen voi piipahtaa muissa haaroissa hallitusti ja nopeasti
